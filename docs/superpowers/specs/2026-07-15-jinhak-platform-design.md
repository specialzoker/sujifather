# 진학 플랫폼 — 설계 문서

작성일: 2026-07-15
작성: 정현석(광명북고 진로진학부장) · with Claude

## 1. 목적

정현석 선생님이 만든 여러 진학/학습 웹앱을 하나로 묶는 **대표 허브 페이지**를 만든다.
현재 네 개의 앱이 각각 다른 주소에 흩어져 있어, 학생·학부모·교사가 한곳에서 찾아 들어갈 입구가 필요하다.
앞으로 앱이 계속 늘어날 것을 전제로, 목록에 한 줄만 추가하면 확장되는 구조로 만든다.

## 2. 범위

### 만드는 것
- 정적 허브 페이지 **한 장**(`index.html`) — 앱들로 연결해주는 대표 입구.
- 기존 앱은 수정하지 않고 **링크로만** 연결한다.

### 넣지 않는 것 (YAGNI)
로그인, 검색창, 방문자 통계, 다크모드, 앱 간 데이터 연동. 지금은 불필요. 필요 시 나중에 추가.

## 5. 디자인 톤

- **딥그린 아카데믹**: 진초록(#24503a 계열) + 크림 배경(#f5f6ee). 따뜻하고 차분한 학교/상담실 느낌.
- 앱 줄(타일): 크림빛 카드 + 옅은 테두리 + 은은한 그림자, 초록 아이콘 블록.
- 분류 소제목: 초록색, 작은 대문자/굵게, 밑줄 구분선.

## 6. 기술 방식

- **순수 HTML/CSS 한 파일.** 프레임워크·빌드 도구 없음. 허브는 링크 페이지이므로 이 방식이 가장 단순·안정적이며, 누구나 열어서 수정 가능.
- **반응형**: 모바일에서도 한 줄 카드가 자연스럽게 보이도록. 상대 단위 사용.
- 외부 의존성 없음(폰트·아이콘은 시스템 폰트/인라인 SVG 또는 이모지로 자급).

## 7. 확장성

앱 목록을 **HTML 안의 간단한 데이터 구조**(예: `<script>`의 배열 또는 반복 가능한 마크업 패턴)로 관리한다.
새 앱이 생기면 **이름·한 줄 설명·주소·분류**만 추가하면 목록에 반영된다. 새 분류도 자유롭게 추가 가능.

## 8. 성공 기준

- https://specialzoker.github.io/ 에 접속하면 허브가 뜨고, 네 앱 링크가 모두 정상 동작(새 탭)한다.
- 모바일·데스크톱 양쪽에서 레이아웃이 깨지지 않는다.
- 새 앱을 추가할 때 목록 데이터에 한 항목만 넣으면 되는 구조다.
